Cable Incremental Twist Unwrapping (Not Supported)

Status note for large multi-turn cable twist paths that cannot be represented by final principal quaternions alone.

Statusnot supported
Affected caseslarge multi-turn endpoint drive and final 27-turn path validation
Current validated alternativesmall/principal twist, initialized material-frame demos, and shape-envelope checks

What Is Missing

Newton can initialize and render large material-frame twist paths, but the current cable validation stack does not yet maintain a path-continuous unwrapped twist coordinate through large multi-turn endpoint or material-frame evolution. A final SO(3) quaternion state is principal-branch limited: many full turns can map back to the same endpoint orientation.

This is why the localized-buckling report validates the quasistatic centerline envelope, not final 27-turn twist retention. The old final twist-transfer panel was removed because it mixed a real-valued initialized twist path with a final principal-quaternion diagnostic.

Video Checks

These videos are included for context: archived twist-unwrap release prototypes (from the twist-unwrap report) and a supported twist-transfer check. They are not evidence that live incremental, multi-turn twist unwrapping is supported.

Expected Future Work

A supported version should carry an incremental/path-continuous twist state or equivalent winding counter, update it during endpoint rotation and solver steps, and report final twist/writhe quantities from that state rather than from a branch-limited quaternion difference. Until then, large multi-turn twist-retention plots should be treated as unsupported.
