Deferred Target

This page intentionally stays in the report as archived design evidence. It documents the live incremental twist-unwrapping idea: raw quaternion twist is branch-limited, so a driven endpoint can appear to lose integer turns unless the solver tracks a path-continuous twist state. That live state is not supported in the active code branch. The current implemented path for prescribed high-turn material winding is rest_twist_total / joint_cable_rest_strain_offset.

The active geometric bend/twist reports validate prescribed material winding and transported-frame twist. This archived page records the separate future problem of live endpoint-driven winding through repeated quaternion branches.
