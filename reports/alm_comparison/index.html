Main tuning conflict

Stack stability prefers moderate stiffness; heavy impact needs much larger stiffness. A single global ke is therefore not a good long-term user-facing enforcement control.

Analysis Notes

  • The contacts example now supports separate VBD box, ball, and static contact stiffness overrides, so the combined scene can avoid one global ke.
  • The isolated stack, isolated ball/ramp, and combined pyramid-plus-ball checks all instantiate the existing example_pyramid with different scene arguments; the combined scene is not a separate custom reconstruction.
  • XPBD iteration count is explicit in the report. The old implicit value was 2 iterations per substep; 1x10 XPBD still fails the 600-frame full-stack drift threshold.
  • The low-budget sweep separates 600-frame pass/fail from 1200-frame confirmation. XPBD 3x2 passes at 600 frames but fails the longer confirmation, so the fastest confirmed XPBD row in that sweep is 8x1.
  • Old soft penalty's fastest confirmed full-stack row is 8x1 with legacy_scale=0.1. 1x4 with legacy_scale=3e-5 short-passes, then fails the 1200-frame run with large drift and negative minimum height.
  • Old hard ALM and most compliant ALM variants confirm at 4x1. Compliant ALM with history and alpha 0 confirms at 3x1 using default rho and ke=3e6.
  • Across the split confirmation, substep-heavy splits such as 4x1 were consistently more reliable than iteration-heavy splits such as 1x4 for the full contact stack.
  • The low-substep/high-iteration sweep shows where more iterations do help: old hard/history confirms at 1x10, new ALM alpha 0 confirms at 2x4 without history and 2x2 with history, while alpha 0.95 variants need 3x2.
  • The strict 2400-frame settlement pass is harder than the 1200-frame confirmation. XPBD needs 4x8 to settle, old soft did not settle in the tested probes, and old hard/history plus all new ALM variants settle at budget 4.
  • For old soft, increasing iterations at one substep did not fix the stack; the long confirmations showed contact-buffer overflow and large drift. Higher substep probes also drifted far before becoming quiet.
  • The targeted stack sweep checks exactly 1x10, 2x5, and 5x2. VBD 1x10 works once legacy scale or compliant-ALM rho is tuned, but alpha 0.95 1x10 rows failed a stricter 1200-frame spot check.
  • rigid_contact_rho is exposed in the example. It can rescue some exact-budget rows such as 1x10, but the fastest confirmed split recommendations prefer default rho when available.
  • New ALM is measured as four combinations: no history/history crossed with alpha=0/alpha=0.95.
  • Same-ke soft penalty cases now set the same explicit shape material ke as the matched ALM row, while also reporting the equivalent legacy scale.
  • Legacy contact stiffness scale is dimensionless: the example applies ke = legacy_scale * cube_mass / sim_dt^2. Changing substeps changes sim_dt, so the same scale implies a different physical ke.
  • Old hard ALM with legacy_scale=1.0 is not equivalent to the new compliant ALM impact row in this run; the isolated ball/ramp test still exceeds the clearance threshold.
  • For the full stack, same-ke soft penalty remains stable but generally has larger horizontal drift than the matched compliant ALM row.
  • For the isolated ball/ramp path, same-ke soft penalty also passes once it uses the ALM-selected ke=3e12, which corresponds to a very large legacy scale.
  • Combined pyramid-plus-ball runs are shown twice: one row uses fastest no-ball stack parameters for pyramid boxes, and another row uses the Full Contacts Stack parameters; both use impact-selected ball/static stiffness.
  • The cable pile default on this branch is compliant ALM with history and passes. Legacy soft and legacy hard/history also pass when copied from main with shape_ke=1e5.
  • Cable bundle hysteresis now appears beside twist and Y-junction, using Dahl hysteresis enabled in both old and new solver-path videos.
  • The selected ke, pass/fail state, and penetration metrics for each combination are listed in the measured table.
  • Contact history controls lambda/contact continuity. It is not the same thing as alpha, and neither one removes the need to tune physical stiffness for the scenario.
