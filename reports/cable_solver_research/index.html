1   The premise: local VBD hits a wall

Vertex Block Descent updates each body using only its immediate neighbors, so a constraint correction propagates along a chain at exactly one body per outer iteration. On an N-body cable the residual scales as O(K/N) and stalls once the local Gauss-Seidel sweep has damped the high-frequency error — increasing K past that stall point yields no further improvement.

Witness scene: long_cantilever_1024_superbend500. 1024 bodies, one end fixed, stretch stiffness 1 × 105, bend stiffness 500 ×  nominal (2500). At this stiffness the correct answer is a chain that hangs nearly straight; local VBD leaves it kinked at ∼4.4° per joint at any K.

Architectural insight. Sparse-direct VBD reaches the exact same residual as local VBD (stretch = 8.71e-3, bend = 0.0765). The wall is not a solver limitation — it is a formulation limitation. Both VBD variants encode cable joints as a penalty k JTJ, so the achievable residual floor is fixed by the penalty stiffness, independent of how well the linear system is solved. Method S + Thomas-bend uses XPBD projection (constraint manifold projection with compliance α = 1/(kΔt2)) — there is no penalty floor. That is why it can push through and give a dramatically tighter residual for a fraction of the cost.

4.5   AVBD paradigm shift — second wall-breaker discovered

Newton's SolverVBD has a built-in Augmented Vertex Block Descent (AVBD, SIGGRAPH 2025, Giles/Yuksel/Diaz) path controlled by rigid_avbd_beta plus per-joint hard/soft mode. When properly configured for cable joints, AVBD's dual-variable ramp breaks the same penalty wall that stops both local and sparse-direct VBD.

Configuration that made it work: (i) rigid_avbd_beta=1.0; (ii) rigid_joint_linear_k_start/...angular_k_start matched to the cable’s actual stiffness (default 100/10 was orders of magnitude lower than cable ke = 1e4 / 500, causing divergence); (iii) after solver construction, call solver.set_joint_constraint_mode(j, hard=True) for each cable joint — cable joints default to SOFT, and AVBD duals only engage on HARD constraints.

AVBD scales — no wall. K=5 → K=80 improves stretch 15× (3e-3 → 2e-4) and bend 4.5× (0.037 → 0.008). The dual-variable ramp pushes constraint penalty toward infinity while the augmented Lagrangian keeps the solve stable. Same architectural conclusion as XPBD projection: the wall is in the fixed-penalty formulation, not the solver. Either escape route (XPBD projection or augmented Lagrangian) breaks it.

Trade-off between the two winners: AVBD is faster and wins big on stretch; M-S+TB wins on bend. The combination of AVBD-hard on stretch + Thomas-bend on bend is the natural next prototype — expected to be strictly Pareto-optimal.

6   What’s next

  1. Parallel Thomas (cyclic reduction / SPIKE / PCR) for O(log N) depth on very long chains — the biggest remaining speedup opportunity vs sparse-VBD at N=4096+.
  2. Y-junction topology: sparse-scalar Thomas on trees to unlock example_cable_y_junction.py.
  3. General rigid body: adapt the same architecture to arbitrary articulation graphs (closures, stacks, contacts).
  4. Tier-A ideas: PCG + VBD preconditioner, multigrid VBD (MGPBD-style), Chebyshev-accelerated VBD.

7   Methodology

Every solver runs on the same GPU with Warp, identical scene setup, identical dt, identical frame budget. Every method uses one config across scenes — no per-scene parameter tweaks. CUDA graph capture enabled on both baseline (SolverVBD) and candidate (Method S + Thomas-bend) so wall-clock comparisons are launch-overhead free.
