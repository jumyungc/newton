1   The premise: local VBD hits a wall

Vertex Block Descent updates each body using only its immediate neighbors, so a constraint correction propagates along a chain at exactly one body per outer iteration. On an N-body cable the residual scales as O(K/N) and stalls once the local Gauss-Seidel sweep has damped the high-frequency error — increasing K past that stall point yields no further improvement.

Witness scene: long_cantilever_1024_superbend500. 1024 bodies, one end fixed, stretch stiffness 1 × 105, bend stiffness 500 ×  nominal (2500). At this stiffness the correct answer is a chain that hangs nearly straight; local VBD leaves it kinked at ∼4.4° per joint at any K.

2b   Sparse-direct block-Cholesky VBD — reaches the same wall

We also ran SolverVBD(rigid_articulation_solve="block_sparse_joints") — the general-graph sparse-direct block-Cholesky articulation solver — on the same witness scene. Same 60-frame protocol, rigid_articulation_relaxation=0.65. It converges much faster in K than local VBD (as any Newton-family direct solver should) but hits the same residual wall.

Architectural insight. Sparse-direct VBD reaches the exact same residual as local VBD (stretch = 8.71e-3, bend = 0.0765). The wall is not a solver limitation — it is a formulation limitation. Both VBD variants encode cable joints as a penalty k JTJ, so the achievable residual floor is fixed by the penalty stiffness, independent of how well the linear system is solved. Method S + Thomas-bend uses XPBD projection (constraint manifold projection with compliance α = 1/(kΔt2)) — there is no penalty floor. That is why it can push through and give a dramatically tighter residual for a fraction of the cost.

6   What’s next

  1. Parallel Thomas (cyclic reduction / SPIKE / PCR) for O(log N) depth on very long chains — the biggest remaining speedup opportunity vs sparse-VBD at N=4096+.
  2. Y-junction topology: sparse-scalar Thomas on trees to unlock example_cable_y_junction.py.
  3. General rigid body: adapt the same architecture to arbitrary articulation graphs (closures, stacks, contacts).
  4. Tier-A ideas: PCG + VBD preconditioner, multigrid VBD (MGPBD-style), Chebyshev-accelerated VBD.

7   Methodology

Every solver runs on the same GPU with Warp, identical scene setup, identical dt, identical frame budget. Every method uses one config across scenes — no per-scene parameter tweaks. CUDA graph capture enabled on both baseline (SolverVBD) and candidate (Method S + Thomas-bend) so wall-clock comparisons are launch-overhead free. Videos are Newton-rendered mp4 at 1920×1080, 300 frames at 60 fps. Note that the main K-sweep table uses 60-frame metric runs; the current embedded videos are 300-frame qualitative renders, so final-frame residuals can differ because the cantilever keeps moving.
